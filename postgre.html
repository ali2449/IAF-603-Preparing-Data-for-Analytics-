Installing PostgreSQL

This guide provides complete, step-by-step instructions for installing PostgreSQL on Windows, macOS, and Linux (focusing on major distributions like Ubuntu, Debian, and Red Hat-based systems such as RHEL, Fedora, and CentOS). These instructions are based on official sources and assume you're installing the latest stable version unless specified. Always check the official PostgreSQL website for the most up-to-date versions and any security advisories. After installation, a common verification step across platforms is to run psql --version in a terminal or command prompt to confirm the installation.

Prerequisites

A supported Windows version (e.g., Windows Server 2022, 2019, 2016, or 2012 R2, depending on the PostgreSQL version).

Download and Installation

  1. Download the interactive installer from EnterpriseDB: Visit https://www.enterprisedb.com/downloads/postgres-postgresql-downloads and select the appropriate version for your system.
  2. Run the downloaded installer (.exe file). You can choose interactive mode (follow the graphical prompts) or silent mode for automation.
  3. During installation, the wizard will guide you through selecting components, including:
    • PostgreSQL server.
    • pgAdmin (a graphical management tool).
    • StackBuilder (for additional tools like drivers and extensions).
  4. Complete the installation by following the on-screen instructions, including setting a superuser password.

Post-Installation Verification and Configuration

PostgreSQL should start automatically as a service. Open pgAdmin or use the command prompt to connect to the server.

To verify: Open Command Prompt and run psql -U postgres (enter the password set during installation), then type \conninfo to check the connection.

If issues arise, check the installer GitHub for bugs: https://github.com/EnterpriseDB/edb-installers/issues.

PostgreSQL can be installed via Homebrew (command-line) or Postgres.app (GUI-based). Homebrew is recommended for developers.

Using Homebrew

Prerequisites

Homebrew installed (install from https://brew.sh if needed).

Installation

  1. Open Terminal and update Homebrew: brew update.
  2. Install PostgreSQL (latest version): brew install postgresql.
    • For a specific version (e.g., 16): brew install postgresql@16.
  3. Start the service: brew services start postgresql (or brew services start postgresql@16 for a specific version).

Post-Installation Verification and Configuration

Verify: Run psql --version.

Create a database: createdb mydb and connect with psql mydb.

Stop the service if needed: brew services stop postgresql.

Using Postgres.app

Prerequisites

None specific beyond macOS compatibility.

Installation

  1. Download Postgres.app from https://postgresapp.com.
  2. Open the downloaded .app file. It runs from the menu bar and starts a PostgreSQL server automatically.

Post-Installation Verification and Configuration

The server runs while the app is open. Connect using included tools or psql.

Verify: Open Terminal and run psql --version.

Closing the app stops the server.

Instructions vary by distribution. Below are for Ubuntu, Debian, and Red Hat-based systems (RHEL, Fedora, CentOS/Rocky/AlmaLinux). Use sudo for commands requiring privileges.

Prerequisites

A supported Ubuntu version (e.g., 24.04 LTS, 22.04 LTS).

Repository Setup

  1. Install PostgreSQL common: sudo apt install -y postgresql-common.
  2. Run the setup script: sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh.
    • Alternatively, manually: Install curl and ca-certificates, import the key, add the repo to sources.list, and update apt.

Installation

  1. Update packages: sudo apt update.
  2. Install: sudo apt install -y postgresql (latest version).
    • For a specific version (e.g., 16): sudo apt install -y postgresql-16.

Post-Installation Verification and Configuration

Start the service: sudo systemctl start postgresql.

Enable on boot: sudo systemctl enable postgresql.

Verify: sudo -u postgres psql and run \conninfo.

Set a password for the postgres user if needed: sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'newpassword';".

Installation

  2. Install: sudo apt install -y postgresql (latest).

Post-Installation Verification and Configuration

Start the service: sudo systemctl start postgresql.

Enable on boot: sudo systemctl enable postgresql.

Verify: sudo -u postgres psql and check with \conninfo.

Configure as needed, e.g., edit /etc/postgresql/<version>/main/postgresql.conf for settings.

Prerequisites

Access to the PostgreSQL Yum Repository for full versions.

Repository Setup

  1. Visit https://yum.postgresql.org, select your version, platform, and architecture.
  2. Run the provided setup script (e.g., install the PGDG RPM: sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm for RHEL 9).

Installation

  1. For distribution repos (limited versions): sudo dnf install -y postgresql-server.
  2. From PGDG repo (recommended): After setup, sudo dnf install -y postgresql16-server (replace 16 with desired version).

Post-Installation Verification and Configuration

Initialize the database: sudo postgresql-setup --initdb (or /usr/pgsql-16/bin/postgresql-16-setup initdb for specific versions).

Enable and start the service: sudo systemctl enable postgresql.service (or postgresql-16.service), then sudo systemctl start postgresql.service.

Verify: sudo -u postgres psql and run \conninfo.

For Fedora's shorter cycle, use PGDG for stability.
